Overview

Commit Intent Detector is a Visual Studio extension that automatically detects the intent of your code changes on file save. It categorizes commits into Bug Fix, Feature, Refactor, Risky Commit, or Documentation update.

Installation

  1. Open Visual Studio and go to Extensions → Manage Extensions.
  2. Search for Commit Intent Detector in the Online section.
  3. Click Download and restart Visual Studio when prompted.
Tip: After installation, an InfoBar will appear in the main window saying "Commit Intent Detector is now active!"

Configuration

  1. Go to Tools → Options → Commit Intent Detector → General.
  2. Set the backend API URL for commit analysis.
  3. Enable insecure SSL if required by your API.
  4. Adjust the debounce delay to control how frequently files are processed on save.

Usage

Once active, the extension automatically analyzes commits when you save a file in a Git repository:

  • Eligible files are processed; others are ignored.
  • The Git diff is sent to the backend API for intent analysis.
  • The detected intent is displayed in an InfoBar, with an optional "Copy to Clipboard" action.

Troubleshooting

  • If no InfoBar appears, ensure the extension is enabled and the file is in a Git repository.
  • Check the Output window for logs prefixed with [CommitIntent].
  • Verify the API URL is correct and accessible.
